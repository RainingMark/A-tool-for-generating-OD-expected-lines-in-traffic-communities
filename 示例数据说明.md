# 示例数据说明

## OD矩阵数据格式

OD矩阵是一个CSV文件，包含交通小区之间的出行流量数据。

### 格式要求：
- 第一列为交通小区编号（起点TAZ）
- 第一行为交通小区编号（终点TAZ）
- 表格内容为出行流量值
- 对角线值通常为0（同一交通小区内的出行）

### 示例：
```csv
TAZ,53621,53622,53623,53624
53621,0,150,200,80
53622,120,0,180,60
53623,190,170,0,90
53624,70,50,80,0
```

## 交通小区数据格式

交通小区数据是一个Shapefile文件（.shp），包含交通小区的几何信息和属性数据。

### 格式要求：
- 必须包含几何信息（面要素）
- 必须包含唯一的交通小区ID字段（推荐字段名：TAZ、taz、ID等）
- 坐标系建议使用投影坐标系（如EPSG:3857）

### 推荐字段：
- TAZ：交通小区编号
- NAME：交通小区名称
- AREA：面积
- POP：人口
- EMP：就业岗位

## 目标交通小区配置

目标交通小区是一个集合，包含用户关注的交通小区编号。

### 格式要求：
- 使用Python集合或列表格式
- 元素为整数类型的交通小区编号

### 示例：
```python
target_tazs = {53621, 53622, 53623, 53624}
```

## 输出文件

程序会生成一个Shapefile文件，包含以下字段：
- Origin_TAZ：起点交通小区
- Destination_TAZ：终点交通小区
- Flow：出行流量
- Length：期望线长度

## 数据准备步骤

1. 准备OD矩阵CSV文件
2. 准备交通小区Shapefile文件
3. 配置目标交通小区编号
4. 运行程序生成期望线
